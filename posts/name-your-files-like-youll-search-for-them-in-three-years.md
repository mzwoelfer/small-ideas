# [[Name your files like you'll search for them in three years]]

Most file systems let you sort by name. Use it!

#### Sort by name. Always.

The one rule that makes everything else work. Every tip here assumes you can sort a folder alphabetically.

## Prefix folders with two-digit numbers

Alphabetical sort is useless without it. Use numbers to enforce the order you want:

```
00_Meetings
01_Projects
98_Deliverables
99_Admin
```

I push recurring admin clutter to the 90s. Once your team knows admin lives at 99, nobody has to think about it.

## Date your files in YYYY_MM_DD

```
2026_05_10_press_release_summer_event_vFINAL.docx
```

Sort by name and you get chronological order for free. Useful when searching across folders.  
`2025_04` narrows your search fast.
It also makes stalled projects visible: if nothing in a folder moved for three weeks, the dates show it.

## Version your files explicitly

`vFINAL`, `vSENT`, `vNEWSLETTER`, `vINSTAGRAM`, `vPRESS`

Removes the pesky "which draft is the right one?" questions.
The date handles ties if two `vFINAL` versions sneak in, the newer date wins.

## Name files fully, not generically

Not `press_release.docx`. Not `final_v3.docx`.

```
2026_05_10_press_release_summer_event_CLUBNAME_vFINAL
```

The file should be findable in a cross-folder search without opening it. Especially useful when you move it in the wrong folder!

## Use aliases for confidential names

If you share your screen or search results, folder names are visible. Instead of naming a client folder after the actual client, use a vague alias: `fluffy_suits`, `wearable_conviction`.
Vague enough to not leak but memorable that you always know what it is.

---

## Just start, don't overthink your folder structure

Start with a rough structure and let it evolve.
Don't spend three hours designing the perfect folder hierarchy before you have even created a single file.
